GESTION PERSONNEL

Dans ce projet JakartaEE, j'ai développé une application web pour gérer le personnel d'une entreprise de travaux publics.

L'application repose sur plusieurs grandes fonctionnalités :

Authentification :
Seul le chef de chantier peut accéder à l’application après vérification du login et du mot de passe dans une base de données MySQL (bdpersonnel).

Mise en place du projet :
- Création d'un projet PersonnelTP sous NetBeans.
- Intégration des bibliothèques JSTL et MySQL.
- Utilisation de Bootstrap 5 pour rendre les formulaires attractifs.
- Configuration de la base de données à l’aide d’un script fourni.
- Déploiement de la structure projet : dossiers jspf, packages controleur, modelesBean, bdd, formulaires.

Développement technique :

- Création d'une servlet AuthentifServlet pour gérer l'affichage et la soumission du formulaire d'authentification.
- Développement d'une classe Connexion pour se connecter à la base de données.
- Développement d'une classe UtilisateurMysql pour valider les identifiants en base (avec des requêtes préparées pour plus de sécurité).
- Développement d'une classe AuthentifForm pour récupérer les données du formulaire, les chiffrer et authentifier l'utilisateur.

Navigation après authentification :
Une fois connecté, l’utilisateur arrive sur une page de menu proposant trois actions :



- Créer un nouveau salarié (formulaire de saisie).
- Lister les salariés existants.
- Affecter les salariés (chantier, bureau, absence).

DEVELOPEMENT WEB LARAVEL

Dans le cadre de mon apprentissage du développement web avec Laravel, j'ai mis en place un environnement de travail professionnel en utilisant Laravel Homestead.

J'ai commencé par installer Homestead, une machine virtuelle Vagrant optimisée pour le développement Laravel. L'installation passe par la configuration d'un fichier Homestead.yaml où j'ai défini le point de montage (mapping entre un dossier de mon ordinateur hôte et un dossier dans la machine virtuelle). Cela permet de synchroniser automatiquement les fichiers entre ma machine locale et Homestead, ce qui facilite le développement en temps réel sans avoir besoin de transférer manuellement les fichiers.

Grâce à Homestead, j'ai disposé rapidement d'un environnement complet avec Nginx, PHP, MySQL, Composer et Laravel, sans avoir à configurer manuellement chaque service.

Après la configuration de l'environnement, j'ai créé un nouveau projet Laravel et configuré les premiers éléments :

- Définition des routes dans le fichier web.php pour lier les URL aux contrôleurs.
- Création des contrôleurs pour gérer les différentes requêtes utilisateurs.
- Développement des vues Blade pour afficher les interfaces de l'application.

Pour mettre en pratique mes connaissances, j'ai développé une application de To-Do List.

Dans cette application :

- J'ai permis aux utilisateurs d'ajouter, d'afficher et de supprimer des tâches.
- J'ai utilisé Eloquent ORM pour interagir avec la base de données.
- J'ai mis en place des migrations pour créer dynamiquement les tables nécessaires.
- J'ai intégré un script JavaScript pour améliorer l'expérience utilisateur : par exemple, marquer une tâche comme terminée sans recharger la page grâce à la manipulation du DOM et à la gestion des événements.

Ce projet m'a permis de renforcer ma compréhension du fonctionnement du framework Laravel, du modèle MVC, de la gestion des bases de données avec Eloquent, ainsi que de l'intégration de fonctionnalités dynamiques en JavaScript dans un projet PHP moderne.

Gestion de parc informatique avec GLPI

Lors de TP de DSDI, j'ai installé et configuré GLPI (Gestionnaire Libre de Parc Informatique) sur une machine Debian 11 Bookworm, déployée dans VirtualBox.
J'ai créé une infrastructure virtuelle composée de trois machines :
- Un serveur Windows Server 2019
- Un client Windows 10
- Un serveur Debian contenant GLPI

J'ai configuré tout le réseau interne, avec le serveur Windows comme passerelle pour permettre la communication entre les machines.

J'ai également mis en place un Active Directory sur le serveur Windows Server 2019. L'Active Directory est un service d'annuaire permettant de centraliser la gestion des utilisateurs, des ordinateurs, et des ressources du réseau. Grâce à l'AD, j'ai pu organiser et sécuriser les accès au réseau et préparer une intégration avec GLPI.

Enfin, j'ai utilisé GLPI pour effectuer une gestion de tickets :
- J'ai créé différents tickets d'incidents et de demandes.
- J'ai simulé le traitement de ces tickets en tant que technicien et administrateur, en suivant leur cycle de vie (création, attribution, suivi, résolution).
- Cela m'a permis de découvrir le fonctionnement complet du module d'assistance de GLPI.
